# 分镜评定标准

本标准用于评估不同文本模型生成的分镜结果，目标不是比较文采优劣，而是判断哪一版最适合后续输入 AI 视频生成模型。评分重点应放在“能否被视频模型直接理解并稳定生成画面”，而不是文学性或表达华丽程度。

## 一、评定目标

1. 判断分镜是否完整覆盖原剧情关键节点
2. 判断镜头拆分是否清晰、可执行
3. 判断文本是否适合视频生成模型理解
4. 判断角色、场景、动作是否稳定一致
5. 选出最适合进入视频生成流程的版本

## 二、核心评定维度

### 1. 剧情覆盖度

查看分镜是否完整保留了原剧本中的主要事件、关键动作、关键对白和情绪转折。

重点看：

- 是否遗漏重要情节
- 是否把核心冲突拆出来
- 是否保留结尾收束或反转

### 2. 镜头颗粒度

查看每个镜头是否信息单一，是否适合单独生成。

重点看：

- 一个镜头是否只包含一个核心动作或一个核心信息点
- 是否避免把太多内容塞进一个镜头
- 是否便于后续逐镜头生成

### 3. 时序清晰度

查看镜头顺序是否符合剧情发展，动作前后是否连贯。

重点看：

- 是否按合理时间顺序推进
- 是否存在动作跳跃或逻辑倒置
- 是否容易让视频模型误解先后关系

### 4. 视觉可执行性

查看文本内容是否能够直接转成画面。

重点看：

- 是否有明确的人物、场景、动作、光线、景别
- 是否避免抽象、空泛、难以成像的表达
- 是否能直接用于视频生成提示词

### 5. 视频模型可理解性

查看文本是否符合视频模型的理解习惯。

重点看：

- 句子是否简洁明确
- 是否减少歧义和复杂嵌套
- 是否方便模型把文字转为连续镜头

### 6. 角色稳定性

查看同一角色在整个分镜中是否保持一致。

重点看：

- 角色名称是否统一
- 外貌、服装、身份、动作特征是否稳定
- 是否避免同一角色多种叫法混用

### 7. 场景一致性

查看场景设定是否统一清晰。

重点看：

- 地点是否固定
- 时间、天气、光线是否一致
- 是否避免无必要的场景跳转

### 8. 动作可拍性

查看动作是否适合视频生成模型表现。

重点看：

- 动作是否短、明确、可视化
- 是否避免过度复杂或不可实现的动作
- 是否适合实际镜头拆分

### 9. 镜头语言能力

查看分镜是否具有基本的镜头思维。

重点看：

- 是否合理使用大全景、中景、近景、特写等景别
- 是否有节奏意识
- 是否能通过镜头切换推动情绪和冲突

### 10. 后续可转写性

查看分镜结果是否容易进一步转成视频 prompt。

重点看：

- 是否容易提炼成“角色 + 场景 + 动作 + 氛围 + 镜头”的结构
- 是否需要大量人工重写
- 是否适合直接进入视频生成流程

## 三、评分建议

建议每项按 1 到 5 分评分：

- 1 分：明显不符合要求
- 2 分：勉强可用，但问题较多
- 3 分：基本合格，有一定可用性
- 4 分：较好，适合进入后续流程
- 5 分：非常优秀，几乎可直接使用

## 四、推荐权重

如果目标是筛选最适合视频生成的版本，建议权重如下：

- 视觉可执行性：20%
- 视频模型可理解性：20%
- 镜头颗粒度：15%
- 时序清晰度：15%
- 角色稳定性：10%
- 场景一致性：10%
- 动作可拍性：5%
- 镜头语言能力：5%

## 五、最终判断原则

最终选择时，不优先看文采，不优先看句子是否漂亮，而优先看：

1. 是否最容易被视频模型理解
2. 是否最容易拆成稳定镜头
3. 是否最少出现歧义和误读
4. 是否最适合直接进入视频生成环节

# Judge Prompt 版本

你现在是一个专业的“分镜评审员”，你的任务不是评价文采，而是评估一组文本模型生成的分镜结果，判断哪一版最适合后续输入 AI 视频生成模型。

## 评审目标

请从“视频生成可用性”的角度对分镜结果打分，重点判断它是否能被视频模型直接理解、直接拆镜头、直接生成画面。

## 评审标准

请按以下维度逐项评分，每项 1 到 5 分，5 分最高：

1. 剧情覆盖度
- 是否完整保留原剧本关键事件
- 是否遗漏重要动作、情绪转折或结尾收束

2. 镜头颗粒度
- 每个镜头是否只包含一个核心动作或一个核心信息点
- 是否适合逐镜头生成
- 是否避免把太多内容塞进一个镜头

3. 时序清晰度
- 镜头顺序是否符合剧情发展
- 动作前后是否连贯
- 是否避免跳跃和逻辑错位

4. 视觉可执行性
- 是否包含明确的人物、场景、动作、光线、景别
- 是否能够直接转成画面
- 是否适合视频生成提示词

5. 视频模型可理解性
- 文本是否简洁明确
- 是否减少歧义
- 是否符合视频模型的理解习惯

6. 角色稳定性
- 同一角色是否名称统一、外观稳定、动作一致
- 是否避免同一角色多种叫法混用

7. 场景一致性
- 场景设定是否统一
- 时间、天气、光线是否前后一致
- 是否避免不必要的场景跳转

8. 动作可拍性
- 动作是否短、明确、物理上可实现
- 是否适合视频生成模型表现
- 是否避免复杂或不可生成的动作

## 评分原则

请只从“视频生成是否好用”的角度评分，不要从文学性、艺术性、表达优美程度来评分。

## 输出要求

请严格输出 JSON，格式如下：

```json
{
  "scores": {
    "plot_coverage": 5,
    "shot_granularity": 4,
    "temporal_clarity": 5,
    "visual_executability": 5,
    "video_model_readability": 5,
    "character_stability": 4,
    "scene_consistency": 4,
    "action_generatability": 5
  },
  "total_score": 4.75,
  "advantages": [
    "..."
  ],
  "problems": [
    "..."
  ],
  "recommendation": "best_for_video_generation"
}
```

## 判定要求

- 如果某个版本明显更适合视频生成，请给出更高总分
- 如果某个版本文笔很好但不适合视频生成，请不要给高分
- 如果某个版本镜头清晰、动作明确、可执行性强，应优先推荐
- 评分标准以“能否稳定进入视频模型”为最高优先级
